# Atlas 1 — M1 Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap the Atlas 1 repo as a pnpm workspace hosting a Tauri 2 desktop app whose webview renders the existing prototype (`index.html` + `console.css`) unchanged. End state: `pnpm tauri dev` opens a native window displaying the Life Console prototype, on Windows/macOS/Linux.

**Architecture:** Monorepo via pnpm workspaces. Rust backend in `src-tauri/` owns the window and (later) vault FS. Webview is a Vite-built Lit/TypeScript app at `apps/console/` — but in M1 it is a thin wrapper that ships the prototype HTML/CSS as static assets. Placeholder packages (`sdk`, `core`, `ui`) and plugin folders (`tasks`, `journal`, `habits`) are stubbed so M2+ can fill them without touching workspace config. No plugins, no REPL, no command registry yet.

**Tech Stack:** pnpm 9, Node 20+, Rust (stable) with Cargo, Tauri 2.x, Vite 5, TypeScript 5, Lit 3 (stub import only in M1), Vitest (dependency added for M2), Playwright (smoke test).

**Prerequisite reading for the executor:**
- Spec: `docs/superpowers/specs/2026-04-19-atlas-1-v1-design.md` — specifically the **Repository Layout** and **Runtime & Stack** sections.
- Existing prototype files at repo root: `index.html`, `console.css` — these are the assets the Tauri webview must render.
- Tauri 2 setup guide: https://v2.tauri.app/start/

**Windows note:** The user runs Windows 11 with bash available. Forward slashes in paths work in bash; `cmd`/`PowerShell` use backslashes. Every command in this plan is written for bash. The `Write` tool can fail with `EEXIST` on Windows when the parent directory already exists — use `cat > path << ENDOFFILE ... ENDOFFILE` (unquoted delimiter, no `$` or backticks in content) if the Write tool fails. This is documented in the repo's global `CLAUDE.md`.

import { test, expect } from "@playwright/test";

test("console boots and renders the prototype shell", async ({ page }) => {
  await page.goto("/");

  await expect(page).toHaveTitle("Atlas 1 — Life Console");

  // Brand mark is present in the chrome
  await expect(page.locator("#brandMark")).toHaveText("ATLAS·1");

  // Statusline shows the level readout
  const statusline = page.locator("#statusline");
  await expect(statusline).toContainText("LVL");
  await expect(statusline).toContainText("17");

  // Nav tabs present
  const nav = page.locator("#screenNav");
  for (const label of ["boot", "home", "health", "projects", "hobbies", "settings"]) {
    await expect(nav.locator(`button[data-scr]:has-text("${label}")`)).toBeVisible();
  }

  // Home screen is on by default
  await expect(page.locator("#scr-home")).toHaveClass(/on/);

  // Bootstrap log fired
  const messages: string[] = [];
  page.on("console", (msg) => messages.push(msg.text()));
  await page.reload();
  await page.waitForTimeout(500);
  expect(messages.some((m) => m.includes("Atlas 1 · v0.1.0 · booted"))).toBeTruthy();
});
